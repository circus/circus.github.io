Case W05-5

Source
Wehrheim2005
Categories
C1 (primary); C2 (secondary)
Domain
model-driven development
formal multiview refinement
Views
class model A
extended class model C
Artefacts
methods get/put in A
get/put/full in C
Quotes
"The classes model buffers with methods put and get, and class C in addition with a method full querying the contents of the buffer... Can a user of A use C as if it were A?"
Summary
A refined class adds a new operation full, raising a substitutivity question for clients expecting only the abstract interface.
Cause: model extension adds new operations at the lower level, so refinement becomes a subtype-compatibility problem rather than simple behaviour preservation.
As a result, the extended model is only consistent if new operations do not interfere with the behaviour and state assumptions inherited from the abstract class.
